## 💡 使用技巧

### 每日检查流程
```
早上（08:00）
1. 指令:应用状态    ← 确认任务运行
2. 指令:电量        ← 确认电量充足

晚上（19:00）
1. 指令:考勤记录    ← 查看今日打卡
2. 指令:日志查询    ← 检查异常情况
```

### 打卡失败处理
```
收到"打卡失败"邮件后
1. 指令:日志查询 30  ← 查看错误原因
2. 指令:重试打卡     ← 尝试重新打卡
3. 指令:应用状态     ← 确认应用正常
```

### 临时任务调整
```
突然需要加班
→ 指令:添加任务 18:30

公司调整作息时间
→ 指令:修改任务 09:00 08:30

出差暂停打卡
→ 指令:暂停循环
回来后 → 指令:开始循环
```

## 🔒 安全提示

✅ 仅接受以下应用的指令：
- 微信（com.tencent.mm）
- 企业微信（com.tencent.wework）
- QQ（com.tencent.mobileqq）
- TIM（com.tencent.tim）
- 支付宝（com.eg.android.AlipayGphone）

✅ 所有指令都会发送确认邮件  
✅ 60秒内相同指令只执行一次  
✅ 完整的参数验证和错误处理  

## ⚠️ 注意事项

1. **时间格式**：必须使用 `HH:mm`（如 `09:00`，不能是 `9:0`）
2. **参数范围**：
   - 超时时间：10-300 秒
   - 延迟息屏：1-60 秒
   - 日志查询：建议不超过50行
3. **网络要求**：执行指令不需要网络，但发送确认邮件需要
4. **权限要求**：需要授予通知监听权限

## 🆘 常见问题

**Q: 为什么指令没有执行？**  
A: 检查以下几点：
- 是否使用支持的应用发送
- 指令格式是否正确（建议使用"指令:"前缀）
- DailyTask 是否在后台运行
- 是否授予了通知监听权限

**Q: 为什么收不到确认邮件？**  
A: 检查以下几点：
- 邮件配置是否正确
- 网络连接是否正常
- 检查邮箱垃圾箱

**Q: 可以在群聊发送指令吗？**  
A: 可以！只要该应用推送了通知，DailyTask 就能接收。
